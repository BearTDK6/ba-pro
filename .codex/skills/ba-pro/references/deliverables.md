# Deliverables

Choose the minimum artifact set that matches the request.

## Discovery Request

Use when scope is unclear or the team is still understanding the problem.

Minimum output:

- project brief
- stakeholder view
- current-state notes or process analysis
- assumptions and constraints
- risks and open questions

## Requirements Definition

Use when the team needs a decision-ready or build-ready requirement set.

Minimum output:

- business requirements
- functional requirements
- non-functional requirements
- business rules
- stories or use cases
- candidate acceptance criteria

## Change Request

Use when an existing scope item changes after baseline or during delivery.

Minimum output:

- change summary
- impact analysis
- affected requirements or stories
- affected acceptance criteria or UAT items
- recommendation and decision points

## Validation and UAT

Use when business testers need clear coverage.

Minimum output:

- UAT plan
- test scenarios
- pre-conditions and data needs
- expected outcomes
- sign-off or defect tracking path

## Delivery Readiness

Use when release coordination or governance matters.

Minimum output:

- requirement status summary
- UAT status summary
- residual risks
- training or communication impacts
- release recommendation

## Traceability Expectations

- Keep stable IDs for requirements, rules, decisions, and scenarios.
- Link downstream artifacts back to the requirement source.
- Update the matrix or trace notes when scope changes.
