# Hotel Allotment

This is a sample feature pack showing how one feature should be organized.

Use this folder when all documents belong to the same feature or request, such as `hotel/allotment`.

## Suggested Usage

- Put feature intake, scope, and as-is understanding in `01_discovery/`.
- Put BRD, FR/NFR, stories, use cases, and business rules in `02_requirements/`.
- Put RTM, gaps, impacts, and delivery risks in `03_analysis/`.
- Put to-be process, solution outline, and diagrams in `04_solution/`.
- Put UAT scenarios, plan, and defect tracking in `05_uat/`.
- Put release and change readiness notes in `06_delivery/`.
- Put decisions, actions, and RAID items in `07_decisions/`.
- Put screenshots and diagrams in `assets/`.
