# Gói Phân Tích

## Diễn Giải Yêu Cầu

`Room Setup` là bước chuyển từ hotel profile sang inventory có thể bán. Chất lượng dữ liệu room ảnh hưởng trực tiếp đến pricing, search và booking.

## Gap Summary

| Khu vực | Rủi ro nếu chưa làm rõ | Trạng thái đích |
| --- | --- | --- |
| Room data model | Thiếu dữ liệu để pricing hoặc search | Có bộ trường room chuẩn |
| Capacity validation | Dữ liệu phòng sai thực tế | Rule capacity được enforce |
| Clone room | User nhập lặp nhiều lần | Có shortcut tăng tốc tạo room |

## Rủi Ro

| Risk ID | Rủi ro | Tác động | Giảm thiểu |
| --- | --- | --- | --- |
| RISK-01 | Không chốt media tối thiểu | QA và UI hiểu khác nhau về completed state | Chốt rule media tối thiểu cho room |
| RISK-02 | Clone room clone sai trường | Tạo dữ liệu room không nhất quán | Xác định rõ trường được clone |

## Khuyến Nghị

- Tách room list và room detail editor rõ ràng.
- Cho phép save draft từng room.
- Dùng validation real-time cho capacity.
