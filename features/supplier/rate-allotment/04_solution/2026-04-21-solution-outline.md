# Phác Thảo Giải Pháp

## Mục Tiêu

Thiết kế step `Rate & Allotment` để supplier tạo rate hợp lệ với ít friction, đồng thời hiểu rõ logic booking mode.

## Cấu Trúc Đề Xuất

1. Rate setup
2. Cancellation policy
3. Optional season / surcharge / promotion
4. Allotment và booking mode summary

## Ghi Chú UX

- Booking mode summary phải luôn hiển thị sau khi user tác động tới allotment.
- Cancellation editor nên hỗ trợ nhìn được timeline các khoảng thời gian.
- Optional sections nên có khả năng skip rõ ràng.
