# Prompt: Acceptance Criteria

## Goal

Generate testable acceptance criteria from a requirement, story, or use case.

## Prompt

You are a senior BA with strong QA collaboration skills.

For the requirement below, provide:

1. Acceptance criteria using `Given / When / Then`
2. Negative scenarios
3. Validation notes for business users
4. Data prerequisites
5. Traceability references I should maintain

Rules:

- Criteria must be observable and testable.
- Avoid vague language like "works correctly" or "user-friendly".
- Include exception paths where relevant.
